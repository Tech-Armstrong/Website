# 002 — Marketing Site Redesign ("Quiet Authority")

**Status:** Proposed — for Cursor execution.
**Scope:** Whole marketing site (home, about/process pages, service & segment pages, knowledge hub, blog chrome, global header/footer).
**Goal:** Read as trustworthy, expert, and discreet — modern and uncluttered — while cutting content to what's direct and scannable. *Less, but sharper.*
**This document is the single source of truth for the redesign.** Cursor should be able to execute it without making further design decisions. Where a real-world fact is required and not in the repo, it is marked `⟦PLACEHOLDER⟧` and listed in §8.

> Companion docs: design tokens live in [`src/app/globals.css`](../../src/app/globals.css); conventions in [design-system.md](../design-system.md); per-task guardrails in [ui-change-checklist.md](./ui-change-checklist.md). Fill out the UI checklist per phase before editing.

## 0. How the site is actually built (so the plan maps to real files)

- **Stack:** Next.js App Router, Tailwind v4 (tokens via `@theme inline` in `globals.css` — there is **no** `tailwind.config`), shadcn/ui conventions, framer-motion via [`MotionProvider`](../../src/components/ui/MotionProvider.tsx).
- **Home** is hand-built from `src/components/home/*` and `src/data/home.ts`.
- **About-us** is its own route (`src/app/about-us/page.tsx`) using `src/components/about-us/*` + `src/data/about-us.ts`.
- **Our-accolade** is its own route (`src/app/our-accolade/page.tsx`) + `src/data/our-accolade.ts`.
- **Everything else** (all services, segments, about-process pages, knowledge hub, careers, contact, FAQs) is **data-driven** through one renderer: `src/app/[slug]/page.tsx` → [`MarketingPageRenderer`](../../src/components/marketing/MarketingPageRenderer.tsx) → [`MarketingPageShell`](../../src/components/marketing/MarketingPageShell.tsx), reading the structured config in **`src/data/marketing-pages/pages.ts`** (auto-generated by `scripts/extract-marketing-content.mjs`).
- The raw `content/pages/*.json` files are **WordPress/Elementor exports** and are **not rendered**. Ignore them as a design surface; the live copy is in `src/data/marketing-pages/pages.ts` and the `src/data/*.ts` files.

> ⚠️ **Generated file caveat:** `src/data/marketing-pages/pages.ts` carries a "do not edit by hand" header and is produced by `scripts/extract-marketing-content.mjs`. Copy rewrites to marketing pages must either (a) edit the generator/its source and re-run it, or (b) deliberately take ownership of the file and remove the header. Decide this in Phase 0 — see §6 and §8-Q1.

## 2. Target design direction — "Quiet Authority"

**Tone:** sophisticated, restrained, trustworthy. Neutral foundation + **one** confident accent (the existing blue, made to work harder). Generous whitespace, strong type hierarchy, quality over decoration. Cut gimmicks (hero typewriter, outline-stroke text, scroll-jacking pin) in favor of calm, deliberate motion.

### 2.2 Typography

**Body / UI: keep Mulish** (`font-body`). Proven, readable, already loaded.

**Headings — primary recommendation: introduce a refined serif display, `Fraunces` (variable), as `font-display`; retire Urbanist.**
Justification: a high-contrast "old-style" serif signals heritage, editorial calm, and fiduciary seriousness — exactly the "discreet, expert, trustworthy" register the brief asks for — while a clean sans body keeps it modern and uncluttered. The serif/sans contrast also creates the strong hierarchy the current all-geometric system lacks. Use Fraunces with optical sizing on large display only; keep weights ≤600 (no heavy/black) to stay restrained. Eyebrows, buttons, nav, and numerals stay in Mulish (or a tight sans) so UI doesn't feel "bookish."

**Fallback (if the client rejects a serif): keep a grotesque, but upgrade Urbanist → `Schibsted Grotesk` (or retain Urbanist) and simply apply the tightened scale below.** Either path uses the same scale and the same `font-display` variable, so component code does **not** change — only the `next/font` wiring in `layout.tsx`. This is flagged for client confirmation in §8-Q3; **default to Fraunces** so Cursor can proceed.

Wiring (in `layout.tsx`, mirroring the current Urbanist/Mulish setup): load `Fraunces` → `--font-urbanist` **variable name kept** (or rename the CSS var to `--font-display` and update the one `@theme inline` line in `globals.css`). Keeping the variable name avoids touching `font-display` consumers.

**Type scale (role-based, applied via `font-display`/`font-body`):**

| Role | Size / leading / weight | Notes vs today |
|------|------------------------|----------------|
| Hero H1 | `clamp(2.5rem, 5vw, 3.75rem)` / 1.05 / 600 | Was 64px hard-stepped; now fluid, **and it's an `<h1>`** |
| Page H1 (inner) | `clamp(2rem, 4vw, 2.75rem)` / 1.1 / 600 | **Promote marketing hero `<h2>`→`<h1>`** (see §6) |
| Section H2 | `clamp(1.75rem, 3vw, 2.25rem)` / 1.15 / 600 | Unifies the 28–44px spread |
| Card / sub H3 | `1.125rem`–`1.25rem` / 1.3 / 600 | Keep |
| Eyebrow | `0.75rem` / 0.12em tracking / 700 uppercase, `--brand-blue` | Keep (Mulish, not serif) |
| Lead paragraph | `1.125rem` / 1.6 / 400 | For hero/section intros |
| Body | `1rem` / 1.7 / 400, `--brand-muted` | Keep |
| Meta / caption | `0.875rem` / 1.5 / 500, `--brand-muted` | Keep |

### 2.3 Motion (framer-motion — purposeful only)

- **Keep:** `ScrollReveal` (opacity + 28px settle), `lift-card`/`hover-lift`, header shell transition, dropdown/mobile-nav transitions, `--ease-settle`/`--dur-*` tokens, full `prefers-reduced-motion` fallbacks.
- **Remove/replace:** hero **typewriter** and **outline-stroke** (replace with a single quiet fade/settle on a static headline); the Why-Choose **scroll-pin** (replace with a calm, non-pinned stepper/accordion — see §4); auto-advancing carousels keep auto-advance **off by default on services** (let the reader drive) — pause-on-hover and honor reduced-motion (already partly done).
- **Principle:** one entrance per section, ≤480ms, ease `--ease-settle`. Motion guides the eye down the page; it never blocks scroll or hides content.
